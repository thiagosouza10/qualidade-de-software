# 🃏 Planning Poker (Scrum Poker)


## 📌 O que é?
Técnica colaborativa de estimativa usada em times ágeis, onde cada membro estima uma tarefa usando cartas (geralmente sequência de Fibonacci).


## 🎯 Objetivo
Alinhar entendimento da tarefa e reduzir vieses durante a estimativa.


## 🧩 Como funciona
1. O time lê a história  
2. Todos escolhem uma carta (ex: 1, 2, 3, 5, 8)  
3. Revelam ao mesmo tempo  
4. Se houver divergência → discussão  
5. Nova rodada até consenso  


## 📊 Exemplo
**História:** Criar automação de login  

- Dev: 3  
- QA: 8  

👉 QA explica riscos (validação, erros, ambiente)  
👉 Time reavalia e chega em 5  


## ✅ Vantagens
- Evita influência entre pessoas  
- Promove discussão técnica  
- Considera diferentes perspectivas  


## ❌ Desvantagens
- Pode levar mais tempo  
- Depende da maturidade do time

# 📏 Story Points


## 📌 O que é?
Unidade de medida usada para estimar esforço, complexidade e incerteza de uma tarefa.


## 🎯 Objetivo
Medir complexidade relativa, não tempo.


## 🧩 O que considerar
- Complexidade técnica  
- Esforço  
- Risco  
- Incerteza  


## 📊 Exemplo
- Task A = 2 pontos (simples)  
- Task B = 5 pontos (média)  
- Task C = 13 pontos (complexa)  


## 💡 Em QA
Inclui:

- Criação de testes  
- Automação  
- Execução  
- Investigação de bugs  


## ✅ Vantagens
- Não prende a tempo fixo  
- Melhora previsibilidade com histórico  


## ❌ Desvantagens
- Difícil no início  
- Pode ser abstrato para iniciantes

# 👕 T-Shirt Size (P, M, G, GG)


## 📌 O que é?
Técnica de estimativa de alto nível usando tamanhos de camiseta.


## 🎯 Objetivo
Fazer estimativas rápidas quando há pouca informação.


## 🧩 Escala comum
- P → pequeno  
- M → médio  
- G → grande  
- GG → muito grande  


## 📊 Exemplo
- Ajuste simples de UI → P  
- Nova funcionalidade → G  
- Integração complexa → GG  


## 💡 Em QA
- P → teste simples  
- M → fluxo com validações  
- G → automação + cenários  
- GG → integração + alto risco  


## ✅ Vantagens
- Rápido  
- Fácil de usar  
- Ideal para backlog inicial  


## ❌ Desvantagens
- Pouca precisão  
- Não detalha esforço real  


## 🧠 Dica final
Use T-Shirt Size no início e depois refine com técnicas como Planning Poker ou estimativa de 3 pontos.
