# 📌 Qualidade de Software - Guia Prático

Este repositório tem como objetivo reunir conceitos, práticas e técnicas essenciais de **Qualidade de Software (QA)**, com foco em testes, revisão e métricas que apoiam a entrega contínua de valor com alta confiabilidade.

---

## 🎯 Objetivo

Centralizar boas práticas de QA para:

* Melhorar a qualidade das entregas
* Reduzir bugs em produção
* Aumentar a eficiência do time
* Promover cultura de qualidade (Shift Left)

---

## 🧪 Técnicas de Teste

### 🔹 Testes Funcionais

* Teste de Caixa Preta
* Particionamento de Equivalência
* Análise de Valor Limite
* Tabela de Decisão
* Teste Baseado em Casos de Uso

### 🔹 Testes Não Funcionais

* Performance (carga, stress)
* Segurança
* Usabilidade
* Compatibilidade

### 🔹 Testes Automatizados

* Testes de API (ex: SuperTest, Postman)
* Testes E2E (ex: Cypress, Playwright)
* Testes Unitários (ex: Jest, Mocha)

---

## 🔍 Técnicas de Revisão

### 🔹 Code Review

* Revisão de lógica e legibilidade
* Verificação de padrões de código
* Identificação de bugs antecipados

### 🔹 Test Case Review

* Validação da cobertura de cenários
* Clareza e objetividade dos testes
* Reutilização e padronização

### 🔹 Revisão de Requisitos

* Identificação de ambiguidades
* Critérios de aceitação bem definidos
* Alinhamento com regras de negócio

---

## 📊 Métricas de Qualidade

### 🔹 Métricas de Teste

* Cobertura de testes (%)
* Taxa de falha
* Defeitos por funcionalidade

### 🔹 Métricas de Processo

* Lead Time
* Cycle Time
* Taxa de retrabalho

### 🔹 Métricas de Defeitos

* Defeitos em produção
* Severidade dos bugs
* MTTR (Mean Time To Repair)

---

## 🚀 Boas Práticas

* Aplicar **Shift Left Testing**
* Automatizar testes repetitivos
* Integrar testes ao pipeline de CI/CD
* Testar desde o início do desenvolvimento
* Trabalhar com TDD / ATDD quando possível

---

## 🛠️ Ferramentas Sugeridas

* Cypress / Playwright (E2E)
* Jest / Mocha (Unitário)
* SuperTest (API)
* Postman / Insomnia
* GitHub Actions / Jenkins (CI/CD)
